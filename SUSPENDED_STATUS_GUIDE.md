# Fly.io Suspended 상태 해결 가이드

## Suspended 상태란?

"Suspended" 상태는 Fly.io 앱의 머신(컨테이너)이 중지된 상태를 의미합니다.

### 주요 원인:

1. **자동 중지 설정 (`auto_stop_machines = true`)**
   - 트래픽이 없을 때 비용 절감을 위해 자동으로 중지됨
   - 요청이 오면 자동으로 다시 시작됨 (`auto_start_machines = true`)
   - `min_machines_running = 0` 설정으로 인해 모든 머신이 중지될 수 있음

2. **헬스 체크 실패**
   - 헬스 체크가 계속 실패하면 머신이 중지될 수 있음
   - 현재 상태: `0/1` (헬스 체크 실패)

3. **수동 중지**
   - 사용자가 직접 중지한 경우

## 현재 상황 분석

화면을 보면:
- ✅ 2개의 머신이 있음
- ❌ 둘 다 `0/1` 헬스 체크 실패
- ⚠️ 모두 Suspended 상태

이는 **헬스 체크가 실패해서** 앱이 제대로 시작되지 않은 것 같습니다.

## 해결 방법

### 방법 1: 머신을 수동으로 시작 (즉시 해결)

1. **Machines 페이지에서:**
   - 각 머신 오른쪽의 **재생 버튼 (▶)** 클릭
   - 또는 머신 이름을 클릭 → 상세 페이지 → `Start` 버튼

2. **또는 Overview 페이지에서:**
   - `Start` 또는 `Resume` 버튼 클릭

3. **머신 시작 후:**
   - 로그를 확인하여 앱이 정상적으로 시작되는지 확인
   - 헬스 체크가 통과하는지 확인

### 방법 2: 헬스 체크 문제 해결 (근본 해결)

현재 `0/1` 헬스 체크 실패는 다음을 의미합니다:
- 앱이 시작되지 않았거나
- `/` 경로에서 응답하지 않거나
- 헬스 체크 타임아웃이 너무 짧을 수 있음

**확인 사항:**

1. **로그 확인:**
   - `Logs & Errors` 탭에서 앱 시작 로그 확인
   - 오류 메시지가 있는지 확인

2. **헬스 체크 설정 확인:**
   - `fly.toml`에 `[[http_service.checks]]` 섹션이 있는지 확인
   - 타임아웃과 grace_period가 충분한지 확인

3. **앱이 실제로 시작되는지 확인:**
   - 로그에서 `Running on http://0.0.0.0:8080` 같은 메시지 확인

### 방법 3: 재배포 (권장)

헬스 체크 설정이 GitHub에 푸시되었지만 아직 배포되지 않았을 수 있습니다:

1. **Overview 페이지로 이동**
   - 왼쪽 메뉴에서 `Overview` 클릭

2. **Deploy 버튼 찾기**
   - 페이지 상단에 `Deploy` 또는 `Deploy latest` 버튼이 있을 수 있음
   - 또는 `Activity` 탭에서 최근 배포를 확인

3. **또는 GitHub에서 자동 배포 확인**
   - GitHub 저장소가 연동되어 있다면 최신 커밋에서 자동 배포 확인

### 방법 4: fly.toml 설정 수정 (자동 중지 방지)

머신이 자동으로 중지되지 않도록 하려면:

```toml
[http_service]
  auto_start_machines = true
  auto_stop_machines = false  # true → false로 변경
  min_machines_running = 1    # 0 → 1로 변경 (최소 1개 머신 유지)
```

**주의:** 이렇게 하면 비용이 발생합니다. 트래픽이 많지 않다면 `auto_stop_machines = true`를 유지하는 것이 좋습니다.

## 즉시 해야 할 일

1. **✅ 머신 시작 버튼 클릭** (가장 빠른 해결)
   - Machines 페이지에서 각 머신의 ▶ 버튼 클릭

2. **✅ 로그 확인**
   - `Logs & Errors` 탭에서 오류 메시지 확인

3. **✅ 헬스 체크 설정 확인**
   - GitHub에 최신 `fly.toml`이 푸시되었는지 확인
   - `[[http_service.checks]]` 섹션이 포함되어 있는지 확인

## 예상되는 결과

머신을 시작한 후:
- 앱이 시작되면 헬스 체크가 `1/1`로 변경됨
- "Suspended" 상태가 사라지고 정상 상태로 변경됨
- `https://jouleheatingsimulation-2d.fly.dev/` 접속 가능

## 문제가 계속되면

1. **로그의 오류 메시지 확인**
2. **앱이 포트 8080에서 제대로 시작되는지 확인**
3. **헬스 체크 경로(`/`)가 올바르게 응답하는지 확인**
